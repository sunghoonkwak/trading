# 스케줄러 서비스 문서 (Scheduler Service)

**스케줄러 서비스** (`scheduler_service.py`)는 트레이딩 시스템 내의 주기적인 작업을 자동화합니다. 주된 역할은 일일 포트폴리오 요약을 생성하고 전송하는 것입니다.

## 주요 책임 (Core Responsibilities)

1. **일일 포트폴리오 스냅샷**: 현재 포트폴리오 상태를 캡처하여 저장합니다.
2. **과거 데이터 비교**: 현재 성과를 1일 전, 1주 전, 1달 전의 과거 데이터와 비교 분석합니다.
3. **리포트 생성**: 자산 변동 내역과 상위 등락 종목을 포함한 상세 요약을 포맷팅합니다.
4. **알림 및 백업**: 리포트를 텔레그램으로 전송하고 파일로 보관합니다.

## 주요 함수 (Key Functions)

### `job_daily_7am()`
매일 오전 07:00에 실행되는 루틴입니다.

- **스케줄 (Schedule)**:
  - **화요일 - 토요일**: 데이터를 캡처하고 JSON 스냅샷(`portfolio_YYYYMMDD.json`)으로 저장합니다.
  - **월요일 - 토요일**: 요약 리포트를 텔레그램으로 전송합니다.
  - **일요일**: 작업을 수행하지 않습니다.
- **데이터 저장 (Data Storage)**:
  - 포트폴리오 스냅샷을 `~/KIS_config/portfolio_history/`에 저장합니다.
  - 리포트 텍스트 백업도 동일한 디렉토리에 저장합니다 (`report_YYYYMMDD.txt`).
- **비교 로직 (Comparison Logic)**:
  - 날짜순으로 정렬된 기록 파일을 사용하여 비교 대상(1일 전, 5일 전, 20일 전 - 워킹 데이 기준)을 찾습니다.

### `start_scheduler()`
백그라운드 스케줄러 스레드를 초기화하고 시작합니다.

- `schedule` 라이브러리를 사용하여 `job_daily_7am`을 등록합니다.
- 메인 애플리케이션을 차단하지 않도록 데몬 스레드에서 실행됩니다.

### `get_comparison_stats()`
리포트의 비교 분석 섹션을 생성합니다.

- 총 자산의 변동률(%)을 계산합니다.
- 전일 대비 변동폭이 가장 큰 상위 3개 종목(Top Movers)을 식별합니다.

## 설정 (Configuration)
- **기록 디렉토리 (History Directory)**: `~/KIS_config/portfolio_history/`
- **실행 시간 (Schedule Time)**: 매일 오전 07:00
