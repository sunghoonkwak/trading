# test/menu.md

본 계획서는 `openapi_trading` 애플리케이션의 각 메뉴 옵션에 대한 통합 테스트를 수행하기 위한 가이드를 제공합니다.

## 주요 목표
- 실제 계좌(`kis_api`)를 연동하여 실제 데이터와 연결성 확인.
- US 시장 주문 위주의 테스트 수행 (국내 주문은 제외).
- 주문 전 사용자 확인 단계를 통해 안정성 확보.

## 상세 테스트 항목

### 1. 계좌 정보 조회 (Account Info)
- 통합 계좌 요약 리포트 출력 확인.
- US 포트폴리오 리스트 조회 및 실제 보유 종목 일치 여부 확인.
- 페이지네이션 및 뷰 모드 전환 테스트.

### 2. US 주문 (Place Order - US Only)
- US 즐겨찾기 종목 선택 프로세스 확인.
- 수량 및 가격 입력 로직 테스트.
- **[중요]** 주문 실행 전 전송 파라미터(티커, 수량, 가격, 시장 등)를 출력하고 사용자의 명시적인 승인(`y/n`) 단계를 거침.
- 승인 시에만 `order_overseas` API 호출.

### 3. 미체결 주문 관리 (Manage Open Orders - US 중심)
- 현재 US 미체결 주문 목록 조회 확인.
- 정정/취소 flow 연동 및 실제 API 호출 결과 확인 (사용자 컨펌 포함).

### 4. 주문 로그 삭제 (Clear Order Logs)
- `latest_logs`에서 `_ODR` 접미사가 붙은 항목들이 정상적으로 제거되는지 확인.

## 주의 사항
> [!WARNING]
> 이 테스트는 **실제 계좌**에서 실시간으로 진행되므로, 실제 체결이 발생할 수 있습니다.
> 주문 테스트 시 반드시 최소 수량이나 안전한 가격 범위를 설정하고, 출력되는 파라미터를 꼼꼼히 확인하십시오.

## 검증 방법
- `tests/test_menus.py` 실행 후 단계별 인터랙티브 상호작용 수행.
- 터미널 UI와 API 응답값의 정합성 확인.
